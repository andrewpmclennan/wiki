====== Git commit squashing ======
Created Friday 06 February 2026

Make sure you do a git pull first!

The basic command is:

git rebase -i HEAD~[NUMBER_OF_COMMITS]

Change everything other than the first message to "f"

Then:
git push --force 


Updating when you have a conflict

git reset --hard origin/name_of_remote_branch

Command to look at the global settings and make sure that they are correct

git config -l 

This opens an editor
git config --global -e
